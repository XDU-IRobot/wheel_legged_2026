# 轮腿机器人 待办列表

### 1. IMU 倒地检测实现

**现状**：`input.cc:217` 中 `fall_detected` 恒为 false，`upright_stable` 恒为 true。FSM 的 `kRecoveryFallCheck` / `kRecoverySelfRight` 状态和 `chassis.cc` 的自起立控制力矩已就位，只缺触发源。

**待做**：
- [ ] 在 `input.cc` 中接入底盘 IMU 姿态判断：俯仰/横滚超过安全阈值 → `fall_detected = true`
- [ ] 实现 `fall_detected_hold_ms` 累加（用于 FSM 确认倒地持续时间）
- [ ] 倒地恢复后检测姿态是否回到安全范围 → `upright_stable = true`

### 2. 状态机整改

**现状**：FSM 已有 12 状态，但部分控制路径（倒地自起力矩计算、跳跃力矩选择）散落在 `chassis.cc` 的 `ComputeActuatorTorque()` 中，通过 `fsm_mode` 和 `jump_phase` 字段分支判断，而非完全由 FSM 状态驱动。

**待做**：
- [ ] 检查所有控制分支是否都有对应的 FSM 状态和状态转移
- [ ] 确保每个 FSM 状态的进入/退出行为清晰、无遗漏
- [ ] 清理 `ControlOutput` 中不受 FSM 完整管理的字段（如 `safe_output_required` 未消费）

### 3. 小陀螺平移优化

**现状**：小陀螺平移已有基础实现（`control_loop.cc:319-325`），前进指令经 `cos(gimbal_heading)` 投影到车体系，由 `kSpinTranslationGain` 缩放。

**待做**：
- [ ] 验证投影公式在不同云台航向下的平移正确性
- [ ] 检查侧向平移在小陀螺期间是否已正确处理（当前仅处理纵向 `s_dot_cmd`）
- [ ] 调参 `kSpinTranslationGain`

### 4. 小陀螺进入条件判断

**现状**：当前小陀螺仅判断拨轮/Shift 键触发（`spin_hold`），无条件进入。

**待做**（二选一，待定）：
- [ ] **方案A**：机体速度 < 阈值 **且** 腿摆角 < 阈值才能进小陀螺
- [ ] **方案B**：冻结位移目标值（进入时锁当前位置），速度自然衰减后进入

### 5. 优化上台阶

**现状**：`kStairClimb` 状态仅在 HighLeg 模式下、双腿 theta > 0.5 rad 时触发，通过降低腿长+回正摆角实现。退出到 `kStairClimbDone` 后有俯仰稳定等待期。

**待做**：
- [ ] 调参：`kStairClimbThetaThresholdRad`（进入阈值）、`kStairClimbLegLengthM`（目标腿长）
- [ ] 图传链路键鼠和dr16键鼠链路下加入两个模式使用按键进入，`上一级台阶`和`上二级台阶`，上一级台阶是上去之后就变低腿长，上二级是上去后变高腿长，然后上完第二级变低腿长。dr16遥控器链路下拨轮只用来上二级台阶


### 6. 优化飞坡（中腿长情况）

**现状**：飞坡中腿长时离地检测（`off_ground_in_mid_high_leg`）触发后轮力矩归零且支持力限幅 ±30N。

**待做**：
- [ ] 中腿长飞坡姿态保持优化，考虑不使用限幅支持力，而是限幅气弹簧补偿，让气弹簧更多参与缓冲

### 7. 跳跃逻辑完善

**现状**：跳跃仅支持低腿长（`fsm.cc:262`），单一跳跃参数（下蹲→蹬伸→回收三段式）。

**待做**：
- [ ] **低腿长大跳**：保留现有三阶段跳跃，可选接入测距自动触发跳跃
- [ ] **中腿长小跳**：新增跳跃参数组，目标跳 5cm 坎（较小的 `kJumpPushLegLengthM`）
- [ ] **高腿长不跳**：在 FSM 中阻止高腿长时进入 `kJumpPrep`
- [ ] 在 FSM 中增加 `kJumpPrepMid` / `kJumpPushMid` / `kJumpRecoverMid` 状态或复用现有状态+参数切换

### 8. 测距传感器

**现状**：使用 CAN DYP 超声波（`include/utils/dyp_can.hpp`，CAN ID 0x120），数据已采集到 `wl_debug` 但未用于控制。

**待做**：
- [ ] 后续换为两路串口测距传感器
- [ ] 将测距数据接入跳跃自动触发逻辑（见第7项）
- [ ] 评估是否需要前方测距用于避障（目前打算靠操作手自行判断）

### 9. 细调倒地自起

**现状**：`chassis.cc:364-477` 的 `ComputeActuatorTorque()` 中有三种自起立姿态处理（滚转异常→单腿旋转；俯仰越界→双腿旋转；恢复→零输出）。但缺少卡住检测。

**待做**：
- [ ] **多种倒地姿态覆盖**：前倒、后倒、侧倒分别测试和调参
- [ ] **卡住检测**：自起立超过一定时间（如 1.5s）仍未见姿态改善 → 先抬头（双腿同时向前转到极限）再摆腿
- [ ] 调参：`kLegRecoverThetaDotTarget`（摆腿速度）、目标角度区间（前倒 -4.0~-3.5 rad / 后倒 -2.2~-1.8 rad）
- [ ] 确定自起立完成后正确退出到 `kLowLeg`（`upright_stable` 判据）
- [ ] 把这个功能写进状态机，而不是散落在其他模式中

### 10. 细调卡尔曼滤波测试打滑

**现状**：`include/chassis/state.hpp` 中的 `SpeedEstimator` 使用 2 状态卡尔曼滤波器融合轮速和 IMU 加速度。有零速钳位（轮速和加速度都小时强制归零）。

**待做**：
- [ ] 打滑场景测试：踩弹丸时轮速和加速度的融合表现
- [ ] 调参：`kKalmanQ`（过程噪声）、`kKalmanR`（观测噪声）
