# wheel_legged 键位绑定

本文档按当前代码实现整理，主要依据：

- `app/targets/wheel_legged/input.cc`
- `app/targets/wheel_legged/include/input.hpp`
- `app/targets/wheel_legged/control.cc`

输入来源优先级：

1. VT03 图传键鼠（云台 C 板 CAN 转发，`tc_remote.valid && !tc_remote.tc_from_dr16`）
2. DR16 兜底（VT03 离线时，DR16 会被包装成 `tc_remote`，但语义分支仍走 DR16 兜底逻辑）

注意：代码里保留了 `dr16_parallel` 字段和并行模式逻辑，但当前没有任何键位会切换它，默认值为 `false`。

---

## 1. VT03 图传键鼠

### 1.1 移动

| 键位 | 触发方式 | 功能 |
|------|----------|------|
| W | 电平 | 前进，`forward = +1` |
| S | 电平 | 后退，`forward = -1` |
| A | 电平 | 左平移，`side = -1` |
| D | 电平 | 右平移，`side = +1` |

`W/S` 优先级互斥，`W` 优先于 `S`；`A/D` 优先级互斥，`D` 优先于 `A`。

### 1.2 工作域

| 键位 | 触发方式 | 功能 |
|------|----------|------|
| Q | 上升沿 | 工作域循环：`Disabled -> Standby -> Enabled -> Disabled` |

对应代码状态：

| `domain_state` | `domain_request` | `standby` |
|----------------|------------------|-----------|
| 0 | `kDisabled` | `false` |
| 1 | `kCombat` | `true` |
| 2 | `kCombat` | `false` |

### 1.3 腿长和上台阶

| 键位 | 触发方式 | 功能 |
|------|----------|------|
| C | 上升沿 | 切换普通中腿长；再次按下回低腿长；同时请求重置底盘正方向 |
| G | 上升沿 | 切换 `mid_leg_g` 中腿长，用另一套斜坡/跨越地形参数；再次按下回低腿长 |
| V | 上升沿 | 切换高腿长，1 次上台阶模式；再次按下回低腿长；同时请求重置底盘正方向 |
| B | 上升沿 | 切换高腿长，2 次上台阶模式；再次按下回低腿长；同时请求重置底盘正方向 |

腿长优先级：上台阶完成锁定低腿长 > 高腿长 > 中腿长 > 低腿长。

### 1.4 运动模式

| 键位 | 触发方式 | 功能 |
|------|----------|------|
| Shift | 电平 | 小陀螺保持，按住生效 |
| F | 上升沿 | 跳跃触发；仅在 `mid_leg_g` 模式下有效；按住 Ctrl 时屏蔽 |

`F` 触发跳跃后只发送一次跳跃请求，不会清掉 `mid_leg_hold` 和 `mid_leg_g`；跳跃流程结束后会继续保持 `mid_leg_g` 中腿长请求。当前低腿长跳跃在 FSM 里仍被注释关闭。

### 1.5 云台控制和自瞄

| 输入 | 触发方式 | 功能 |
|------|----------|------|
| 鼠标 X | 增量 | 云台 yaw 目标积分 |
| 鼠标 Y | 增量 | 云台 pitch 目标积分 |
| 鼠标右键 | 电平 | 按住进入自瞄模式 |
| 鼠标左键 | 电平 | infantry 发射手动触发；hero 路径当前未使用左键 |

右键自瞄子模式由 `aim_mode` 决定：

| `aim_mode` | 右键按住时的 `combat_profile` | 发给自瞄的模式值 |
|------------|------------------------------|----------------|
| `kAmmo` | `kAutoAimAmmo` | 1，装甲板 |
| `kFuSmall` | `kAutoAimFuSmall` | 2，小符 |
| `kFuBig` | `kAutoAimFuBig` | 3，大符 |

`kAutoAimFuSmall` 和 `kAutoAimFuBig` 会额外请求 `standby`，也就是打符时底盘保持低腿长姿态但轮子不输出。

### 1.6 自瞄子模式切换

| 键位 | 触发方式 | 功能 |
|------|----------|------|
| R | 上升沿 | `kAmmo → kFuSmall → kFuBig → kAmmo` 循环切换 |

具体逻辑：

- 每按一次 `R`，`aim_mode` 按 `kAmmo → kFuSmall → kFuBig → kAmmo` 循环递增。
- 无力→有力（domain 使能）时自动重置为 `kAmmo`。

### 1.7 摩擦轮和发射相关

| 键位 | 触发方式 | 功能 |
|------|----------|------|
| Ctrl+Z | 上升沿 | 摩擦轮目标转速 `-kFricSpeedStepRpm` |
| Ctrl+X | 上升沿 | 摩擦轮目标转速 `+kFricSpeedStepRpm` |

摩擦轮目标转速首次使用时从 `params::active::shoot::kFricSpeedTargetRpm` 初始化。

### 1.8 UI 和恢复

| 键位 | 触发方式 | 功能 |
|------|----------|------|
| E | 电平 | UI 刷新/重绘使能，按住时生效 |
| Z | 长按 1 s | 切换倒地自启自动/手动模式 |
| A / D | 电平 | 仅在倒地自启手动模式下，控制左/右腿正向摆动 |
| Ctrl+A / Ctrl+D | 电平 | 仅在倒地自启手动模式下，控制左/右腿反向摆动 |

### 1.9 当前采集但未绑定的键

| 键位 | 当前状态 |
|------|----------|
| R | 掩码被采集为 `0x0100`，但没有实际语义处理；`reset_yaw_request` 当前由 `C/V/B` 触发 |

## 2. DR16 遥控器兜底

### 2.1 左拨杆：工作域

| 左拨杆位置 | 功能 |
|------------|------|
| 上 | `kCombat` |
| 中 | `kService` |
| 下 | `kDisabled`；同时与右拨杆组合进入云台测试模式 |

左下 + 右拨杆组合：

| 左拨杆 | 右拨杆 | 功能 |
|--------|--------|------|
| 下 | 上 | `kService` + 云台辨识模式 `kIdent`，腿长低 |
| 下 | 中 | `kService` + 前馈验证模式 `kFfVerify`，腿长低 |
| 下 | 下 | `kDisabled`，腿长低 |

### 2.2 右拨杆：腿长和战斗子模式

非战斗域、且不是云台测试模式时：

| 右拨杆位置 | 腿长 |
|------------|------|
| 上 | 高腿长 |
| 中 | 中腿长 |
| 下 | 低腿长 |

战斗域时：

| 右拨杆位置 | `combat_profile` | 腿长 |
|------------|------------------|------|
| 上 | `kAutoAimFuSmall` | 低腿长 |
| 中 | `kAutoAimAmmo` | 低腿长 |
| 下 | `kNormal` | 低腿长 |

DR16 拨杆没有直接选择 `kAutoAimFuBig` 的路径。

### 2.3 DR16 摇杆、鼠标和键盘

VT03 离线且 DR16 在线时：

| 输入 | 功能 |
|------|------|
| 右摇杆 Y | 前进/后退 |
| 右摇杆 X | 左右平移 |
| 左摇杆 X | 云台 yaw 目标积分兜底 |
| 左摇杆 Y | 云台 pitch 目标积分兜底 |
| 鼠标 X | 云台 yaw 目标积分，优先于左摇杆 |
| 鼠标 Y | 云台 pitch 目标积分，优先于左摇杆 |
| 鼠标右键 | 按住进入自瞄模式 |
| 键盘 W/S/A/D | 移动输入，优先于右摇杆 |
| 键盘 Z 长按 1 s | 切换倒地自启自动/手动模式 |
| 倒地自启手动模式下 A/D | 控制左/右腿正向摆动 |
| 倒地自启手动模式下 Ctrl+A/Ctrl+D | 控制左/右腿反向摆动 |

DR16 键盘的 `Q/C/G/V/B/F/E/Ctrl+F/Ctrl+G/Ctrl+Z/Ctrl+X` 当前不会走 VT03 语义分支。

### 2.4 DR16 拨轮

| 拨轮输入 | 功能 |
|----------|------|
| 拨轮 >= `kWheelSpinThreshold` | 小陀螺保持 |
| 拨轮回到中心附近 | 重新武装跳跃触发 |
| 拨轮 <= `-kWheelActionThreshold` | 触发跳跃；战斗域下屏蔽 |

云台测试模式 `kIdent/kFfVerify` 下，拨轮的小陀螺和跳跃逻辑不生效。

## 3. DR16 并行模式代码保留项

代码里有 `tc_state.dr16_parallel`，如果该值为 `true` 且 VT03 键盘完全空闲，DR16 可并行补充：

| DR16 输入 | 并行行为 |
|-----------|----------|
| 左拨杆 | 仅当 VT03 当前工作域为 disabled 时，覆盖工作域 |
| 右拨杆 | 仅当 VT03 腿长为低腿长时，覆盖腿长 |
| 拨轮上拨 | 补充小陀螺请求 |
| 拨轮下拨 | 补充跳跃请求，战斗域下屏蔽 |
| 右摇杆 | 键盘无移动输入时补充移动 |
| 左摇杆 | 鼠标静止且键盘空闲时补充云台积分 |

当前代码没有键位会把 `dr16_parallel` 切到 `true`，所以正常运行时这段逻辑不会启用。
