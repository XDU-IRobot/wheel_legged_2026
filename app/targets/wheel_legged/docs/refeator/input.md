# Wheel Legged 控制输入与键位映射

## 1. 文档目的

本文档定义 `wheel_legged` 重构后的统一控制输入语义，以及 DR16 遥控器、DR16 键鼠和图传键鼠的键位映射。

输入层应先将不同硬件输入转换为同一份控制语义，再交给底盘、云台、发射机构和状态机。下游模块不得直接解析拨杆、拨轮、键盘位掩码或鼠标原始值。

## 2. 输入源及优先级

系统支持以下输入源：

1. DR16 遥控器拨杆、摇杆和拨轮。
2. DR16 接收机携带的键盘、鼠标输入。
3. 图传链路转发的键盘、鼠标输入。

输入源优先级固定为：

```text
DR16（包括遥控器和 DR16 键鼠） > 图传键鼠 > 无有效输入
```

仲裁规则：

- DR16 在线时，模式、移动、云台和发射语义均以 DR16 数据为准，图传输入不得叠加或覆盖 DR16 命令。
- DR16 离线且图传键鼠在线时，使用图传键鼠。
- 所有输入源均无效时，产生失能请求，系统进入安全状态。
- 输入源切换时必须重新同步按键边沿状态，不能因为来源切换产生虚假的单击事件。
- 同一个控制周期最终只允许产生一份统一控制语义。

## 3. 统一输入语义

建议统一语义至少包含以下内容：

| 分类 | 语义 |
| --- | --- |
| 输入状态 | 来源、是否有效、时间戳 |
| 整车模式 | Disable、Standby、Normal、Fly、Upstairs、Spin、SpecialSpin、Fall、Jump 请求 |
| 底盘移动 | 前后方向、侧向方向、速度倍率 |
| 云台 | 手动 yaw/pitch 增量、自瞄请求、自瞄目标类型 |
| 发射 | 摩擦轮允许、拨盘允许、单次/持续发射请求 |
| 动作事件 | 跳跃、上台阶任务、取消任务、180° 翻转、正方向重置 |
| UI | UI 强制刷新、AD 功能状态 |

持续命令和单周期事件必须分开。例如 `Spin` 是持续电平，而 `Jump`、`Flip180` 是单周期触发事件。

## 4. DR16 遥控器模式映射

### 4.1 拨杆组合

以下为目标映射。`底盘无力` 表示底盘关节电机和轮毂电机不输出力矩，但云台仍可工作；它不等同于整车全部设备关闭。

| 左拨杆 | 右拨杆 | 工作用途 | 底盘 | 云台 | 摩擦轮/拨盘 |
| --- | --- | --- | --- | --- | --- |
| `kDown` | `kDown` | 安全自瞄 | 无力 | Aimbot | 关闭 |
| `kDown` | `kMid` | 静止自瞄射击 | 无力 | Aimbot | 开启 |
| `kDown` | `kUp` | 运动自瞄射击 | 有力，默认 Normal | Aimbot | 开启 |
| `kMid` | `kDown` | 低姿态调试 | Normal / Low | 手动调试 | 关闭 |
| `kMid` | `kMid` | 中姿态调试 | Fly / Mid | 手动调试 | 关闭 |
| `kMid` | `kUp` | 高姿态、上台阶调试 | Upstairs / High | 手动调试 | 关闭 |
| `kUp` | `kDown` | 下台阶模式 | Reserved | 手动 | 关闭 |
| `kUp` | `kMid` | 云台前馈验证 | 无力 | FfVerify | 关闭 |
| `kUp` | `kUp` | 云台辨识 | 无力 | Ident | 关闭 |

补充说明：

- `kUp + kDown` 只保留输入语义和状态入口，当前没有完整的下台阶控制功能；未实现前必须保持安全输出，不得自动进入上台阶逻辑。
- `kUp + kMid` 暂定对应当前代码中的 `GimbalTestProfile::kFfVerify`。
- `kUp + kUp` 暂定对应当前代码中的 `GimbalTestProfile::kIdent`。
- 三种 Aimbot 组合默认沿用当前 NUC 目标有效性判断：NUC 目标有效时使用 Host 目标，否则降级到人工云台目标。
- Aimbot 的装甲板/小符/大符目标类型仍沿用键鼠右键与 G 键逻辑；如果以后需要完全由拨杆确定，应另行修改本表。

### 4.2 摇杆映射

未特别说明的摇杆语义沿用当前代码：

| 输入 | 语义 |
| --- | --- |
| 右摇杆 Y | 底盘前进/后退 |
| 右摇杆 X | 底盘侧向移动/方向选择 |
| 左摇杆 X | 云台 yaw 目标增量 |
| 左摇杆 Y | 云台 pitch 目标增量 |

摇杆输入需要归一化到 `[-1, 1]`，死区、速度上限和斜坡由统一底盘命令生成器处理。

### 4.3 拨轮映射

拨轮使用三个逻辑区间：上拨、回中、下拨。所有单次动作都必须经过回中后才能再次触发。

| 左拨杆 | 右拨杆     | 上拨 | 下拨 |
| --- |---------| --- | --- |
| `kDown` | `kUp`   | `SpecialSpin` | `Spin` |
| `kDown` | `kMid`  | 发射 | 暂无新动作，维持当前安全行为 |
| `kDown` | `kDown` | 发射 | 暂无新动作，维持当前安全行为 |
| `kMid` | `kDown` | 手动 `Jump` | `Spin` |
| `kMid` | `kMid`  | 维持当前逻辑：手动 `Jump` | 维持当前逻辑：`Spin` |
| `kMid` | `kUp`   | 维持当前逻辑：手动 `Jump` | 维持当前逻辑：`Spin` |
| `kUp` | 任意      | 无动作 | 无动作 |

说明：

- `SpecialSpin` 为暂留功能，目前只生成模式请求，不执行变化腿长的变速小陀螺控制。
- `Spin` 为拨轮持续保持型命令；拨轮回中后退出。
- `Jump` 为上拨边沿事件，不应在拨轮持续保持时重复触发。
- Aimbot 射击模式下，上拨表示发射。是否单发、连发及热量限制由发射控制器决定。
- 左中右中、左中右上的拨轮行为没有在新需求中重新指定，因此暂时记录为沿用当前代码；如仅允许低姿态跳跃，应在实现前将这两项改为无动作。

## 5. 键鼠映射

DR16 键鼠和图传键鼠应使用同一套映射，区别只在输入源优先级。

| 键位 | 当前/目标语义 | 类型 |
| --- | --- | --- |
| `W` / `S` | 前进 / 后退 | 持续 |
| `A` / `D` | 侧向移动；AD 功能关闭时屏蔽，Spin 中不屏蔽 | 持续 |
| `Q` | Disable 与 Enabled 切换 | 单击 |
| `Ctrl + Q` | Standby 与 Enabled 切换 | 单击 |
| `Shift` | Spin | 持续 |
| `C` | Normal 与 Fly 姿态切换，并取消当前上台阶任务 | 单击 |
| `F` | Fly 慢速参数模式开关 | 单击 |
| `V` | 单次 Upstairs 任务 | 单击 |
| `B` | 双次 Upstairs 任务 | 单击 |
| `R` | 云台目标旋转 180°，并翻转底盘跟随方向 | 单击 |
| `G` | Aimbot 类型循环：Ammo → FuSmall → FuBig | 单击 |
| `Z` | AD 功能开关 | 单击 |
| `E` | UI 强制刷新 | 持续 |
| `X` | 当前未定义，保留但不产生控制动作 | Reserved |
| 鼠标移动 | 云台 yaw/pitch 目标增量 | 持续 |
| 鼠标右键 | 按住进入 Aimbot，松开退出 | 持续 |
| 鼠标左键 | 发射请求 | 持续 |
| 鼠标滚轮上 | 下台阶模式开关 | 单击，Reserved |
| 鼠标滚轮下 | 低姿态下手动 Jump | 单击 |

当前实现注意事项：图传 CAN 桥中的 `mouse_z` 当前被固定为 `0`，因此图传滚轮动作尚不能真正触发。重构实现该映射前必须恢复滚轮数据解析。

## 6. 安全约束

- 输入离线、裁判系统禁止底盘输出或执行器反馈异常时，安全事件优先于所有模式请求。
- 摩擦轮允许与拨盘允许必须作为明确语义输出，不能仅通过云台状态间接推断。
- 底盘无力、云台无力和整车 Disable 必须是三个可区分的控制权限，避免为了自瞄而误使能底盘。
- Reserved 功能只能产生可观测请求，不得进入未完成的力矩控制路径。
- 输入层只表达驾驶员意图，不直接修改 PID、腿长目标或电机输出。
