# 云台控制工具套件

仓库包含用于云台系统建模、参数识别、前馈设计的一套工具，以及定量评估云台性能指标的一套工具。

## 动力学前馈使用指南

本项目提供了一套基于最小二乘法的云台动力学参数辨识与 C++ 前馈控制部署方案，并支持移动基座的 3D 加速度/重力场补偿。整个流程分为“数据采集 -> 模型辨识 -> 实机部署”三步。

### 0. 准备环境

```bash
pip install -r requirements.txt
```

### 1. 数据采集

1. 首先在云台的STM32上面写一套能让云台位控（电机编码器位控，不是imu）的代码

2. `dynamics/harmonics_generator.hpp` 里面是一个谐波信号发生器，调一下参数让这个东西输出范围尽量大的0~5次谐波信号（但是不要超出云台的限位！），频率0.1~0.3Hz，让云台执行这个信号的位控运动。

3. 按照`dynamics/dynamics.hpp`里面的坐标系定义，把云台的状态数据用串口发出来

  然后在电脑上运行 `dynamics/serial_collect.py`采集云台在闭环运动过程中的状态数据，采集出来的数据会被保存到 `ident_data.csv` 文件里

  CSV 表头必须是 `time, q1, q2, dq1, dq2, tau1, tau2`。

  - `q/dq`: Yaw (1) 和 Pitch (2) 轴的角度 (rad) 和角速度 (rad/s)。
  
  - `tau`: 电机实际输出的力矩（可以通过采集实际电流替代，后续计算出的是电流前馈）。

### 2. 参数辨识
打开 Jupyter Notebook 运行 `dynamics/ident.ipynb` 的所有单元格：
- 代码使用了符号运算库 (`sympy`) 推导严格的拉格朗日刚体动力学，重组了满秩矩阵，最后利用 `sklearn` 读取数据文件计算偏最小二乘。
- 运行完毕后会直接输出 **9 个基参数** (包含复合惯量、偏心质心、粘性摩擦与库仑摩擦) 以及 $R^2$ 拟合优度评分。
- **排错建议**：$R^2$ 越高越好（>0.8 说明极为精准）。如果你发现辨识出的部分惯量参数（`theta_1`, `theta_2` 等）为微小负数，这通常是由于底层控制器的通信与差分延迟导致的高频相移引起的，这不影响低频的重力和摩擦拟合。在部署时你可以手动把这些负惯量置 0。

### 3. 实机部署

辨识出模型参数后，在STM32上进行部署：

1. 把 `dynamics/dynamics.hpp` 类直接复制过来

2. 把 `dynamics.hpp` 里面的私有成员变量 `THETA` 中的 9 个数值替换为你刚才辨识出来的值。

3. 先把前馈计算函数 `ComputeFf` 直接调用一下，看看输出的前馈力矩是否在合理范围内，辨识的不好完全有起飞的可能。如果输出的力矩很明显不对，说明辨识效果不好，也就是说明采集的数据不好，云台在执行谐波信号的时候不要太快，不要给外部干扰，尽量采集多一点的数据。

4. 如果前馈模型输出的力矩看起来差不多，在主控制循环中调用前馈计算函数，把前馈输出和 PID 反馈输出叠加起来，作为最终的电机控制输入。
   ```cpp
   #include "dynamics.hpp"
   
   Gimbal2DofDynamics dynamics;
   
   // ... 在控制循环中 ...
   // 1. 获取当前状态与期望的角加速度
   float ddq1_ref = ... ; // Yaw 期望角加速度
   float ddq2_ref = ... ; // Pitch 期望角加速度
   
   // 2. 传入基座的 IMU 加速度计矢量（如果基座固定，传入 [0, 0, -9.81] 或实际重力分布）
   // 这种 3D 重力场代入可以消除移动机器人（如机械狗、全向车）底盘急加减速给云台造成的甩头干扰
   Eigen::Vector3f g_imu(imu_accel_x, imu_accel_y, imu_accel_z); 
   
   // 3. 计算前馈输出
   Eigen::Vector2f tau_ff = dynamics.ComputeFf(q1, q2, dq1, dq2, ddq1_ref, ddq2_ref, g_imu);
   
   // 4. 前馈 + 反馈 控制
   tau_out_yaw = pid_yaw.calculate() + tau_ff.x();
   tau_out_pitch = pid_pitch.calculate() + tau_ff.y();
   ```

### 4. 评估效果

1. 先把PID关掉，完全只给电机前馈力矩，但是在用ComputeFf函数计算前馈的时候，把dq、ddq全部置零，这样就相当于只用了重力补偿，看看云台能不能稳住不掉下来。如果能稳住，说明重力补偿是成功的，代表辨识出的重力相关参数（偏心质心和重力惯量）是对的。

2. 经过1的初步验证之后，可以把前馈的动态部分也打开，然后用性能评估工具（后续更新）定量分析云台在各频率信号下的跟踪误差，这样可以比较全面的评估前馈控制的效果。
