# Hero 上台阶（Stair Ascent）完整流程分析

## 架构概览

三层架构：
```
用户输入 (input.cc)  →  任务协调器 (stair_task_coordinator)  →  攀爬序列 (stair_climb_sequence)  →  底盘FSM + 控制器
```

---

## 第一层：用户触发（input.cc）

| 触发方式 | 按键/操作 | 请求类型 | 行为 |
|---|---|---|---|
| **V 键** | 键盘 V（边沿触发） | `kArmSingle` | 单步上台阶 |
| **B 键** | 键盘 B（边沿触发） | `kArmDouble` | 双步上台阶 |
| **DR16 高腿开关** | 拨杆切到 kHigh | `kArmContinuous` | 持续模式（自动 Arm） |
| **取消** | C键 / 鼠标滚轮上 / Shift(spin) | `kCancel` | 取消当前台阶任务 |

---

## 第二层：任务协调器（stair_task_coordinator）

**状态机（StairTaskMode）：**
```
kIdle → kArmed → kExecuting → kBetweenSteps → kArmed → ...（循环多步）
                                    ↓
                              kSucceeded → kIdle
                                    ↓
                              kAborted → kIdle
```

**状态转换条件：**
- `kIdle → kArmSingle/kArmDouble/kArmContinuous`：用户请求
- `kArmed → kExecuting`：`contact_released && high_leg_ready && contact_detected`（双足接触阈值 > 0.50 rad）
- `kExecuting → kBetweenSteps`：序列成功且还有剩余步数
- `kExecuting → kSucceeded`：序列成功且所有步数完成
- `kBetweenSteps → kArmed`：`contact_detected == false`（足部释放）
- 任何状态 → `kAborted`：用户取消 / 姿态无效 / 超时 / 输出禁用

**输出标志：**
- `request_high_leg`：kArmed 或 kBetweenSteps 时为 true → 强制 FSM 切换到高腿
- `force_low_leg`：kSucceeded 或 kAborted 时为 true → 强制 FSM 切换到低腿
- `task_active`：kArmed/kExecuting/kBetweenSteps 时为 true

---

## 第三层：攀爬序列（stair_climb_sequence）

**Phase 状态机（StairPhase）：**
```
kIdle → kHook → kRetract → kSettle → kSucceeded
  ↓       ↓        ↓          ↓
  └───────┴────────┴──────────┴→ kAborted
```

### Phase 1: kHook（钩住台阶边缘）
- **目标角度**：双腿 `hook_theta_target_rad = 1.0 rad`（约 57°）
- **腿长**：`hook_leg_length_m = 0.37m`
- **车轮**：**关闭**（纯腿部扭矩控制）
- **稳定时间**：100ms
- **超时**：2000ms
- **动作**：双腿向前摆动，钩住台阶边缘

### Phase 2: kRetract（收缩腿部，拉起身体）
- **目标角度**：双腿 `retract_theta_target_rad = 1.5 rad`（约 86°）
- **腿长**：`retract_leg_length_m = 0.10m`
- **车轮**：**关闭**
- **稳定时间**：500ms
- **超时**：2000ms
- **动作**：双腿大幅后摆并收缩腿长，将身体拉上台阶

### Phase 3: kSettle（稳定）
- **目标角度**：`settle_theta_target_rad = 0.0 rad`（回到中立位）
- **腿长**：`settle_leg_length_m = 0.10m`
- **车轮**：**关闭**
- **稳定时间**：300ms
- **超时**：2000ms
- **动作**：双腿回正，稳定在台阶上方

### Hero 特殊行为
- **腿长来源**：由 FSM 决定（不是序列），control.cc 第 710-712 行有 `#if WHEEL_LEGGED_ROBOT_VARIANT == 1` 的特殊 override
- **kStairExitStandupHookDelayTicks = 0**：台阶退出后无延迟直接 standup

## 底盘 FSM 中的状态转换（chassis_fsm.cc）

```
用户触发 → request.stair_task_request
    ↓
chassis_fsm.cc:422-432
    if (stair_task.request_high_leg) → next_mode = kStairTask
    if (stair_task.force_low_leg) → next_mode = requested_normal_state
    ↓
kStairTask 状态输出（chassis_fsm.cc:202-213）:
  - enable_dm = true
  - spin_enable = false
  - leg_profile = kHigh
  - target_leg_length_m = 0.37m (step1) 或 0.33m (step2)
  - jump_phase = 0
```

---

## 控制循环集成（control.cc）

1. **阶段 2（第 349-401 行）**：计算接触检测、高腿就绪状态，更新协调器 → 更新序列
2. **阶段 7（第 708-713 行）**：注入运动目标到 chassis_update_input
   - Hero 变体：`leg_length_m` 来自 FSM（0.37m/0.33m），theta 来自序列各阶段

## 完整时序图

```
用户按 V 键（单步上台阶）
    │
    ▼
[Input] input.cc → StairTaskRequest(kArmSingle)
    │
    ▼
[Coordinator] kIdle → kArmed
    │  request_high_leg=true
    ▼
[FSM] → kStairTask (leg_length=0.37m, high_leg)
    │
    ▼
[Contact Detection] 等待双腿 theta > 0.50 rad
    │  （机器人向前行驶，碰到台阶）
    ▼
[Coordinator] kArmed → kExecuting
    │  start_sequence = true
    ▼
[Sequence Phase 1: kHook]
    │  双腿摆至 1.0 rad (57°), leg=0.37m, 车轮 OFF
    │  稳定 100ms
    ▼
[Sequence Phase 2: kRetract]
    │  双腿摆至 1.5 rad (86°), leg=0.10m, 车轮 OFF
    │  稳定 500ms — 身体被拉上台阶
    ▼
[Sequence Phase 3: kSettle]
    │  双腿回至 0.0 rad, leg=0.10m, 车轮 OFF
    │  稳定 300ms
    ▼
[kSucceeded]
    │
    ▼
[Coordinator] kExecuting → kSucceeded → kIdle
    │  force_low_leg = true
    ▼
[FSM] kStairTask → kLowLeg (回到正常状态)
```

---

## 中止条件

- 用户按 C 键 / Shift(spin) / 鼠标滚轮上
- 姿态超出安全范围
- 输出禁用（domain disabled 或 standby）
- 任何阶段超时（第一步 2000ms，第二步 1000ms）
- 跌倒检测触发
