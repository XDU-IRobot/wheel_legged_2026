# Input 数据流：本地工作区 vs GitHub (origin/feature/hero) 对比分析

## 概览

当前在 `feature/hero` 分支，本地工作区有大量未提交修改。其中 6 个文件有**实质性内容差异**（其余文件仅为 LF/CRLF 行尾差异）：

| 文件 | 变更性质 |
|------|----------|
| `input.cc` | 重构：单体函数拆分为 4 个子函数 |
| `input.hpp` | 新增结构体 + 删除字段 |
| `control.cc` | 适配新 API |
| `fsm_common.hpp` | 删除手动恢复模式字段 |
| `chassis.hpp` | 删除手动恢复模式字段 |
| `chassis.cc` | 删除手动恢复模式控制逻辑 |

## 主要差异详解

### 1. 架构重构：`ResolveInputSemantics` 拆分为子函数

**GitHub（旧）**：一个 ~500 行的巨型函数，所有逻辑内联。

**本地（新）**：拆分为 4 个独立子函数：
- `ResolveTcKeyboardEdges()` — TC 键盘上升沿检测（C/G/Q/V/B/F/R/Z/E 键）
- `ResolveDr16Mode()` — DR16 拨杆→模式映射（domain/leg/combat/test/jump/spin）
- `ResolveTcMode()` — TC 键鼠→模式映射（domain/leg/spin/jump/stair-descend）
- `IntegrateGimbalTarget()` — 云台 yaw/pitch 目标积分

新增辅助函数：
- `IsAutoAimProfile()` — 判断是否自瞄模式
- `ResolveAutoAimProfile()` — AimMode → CombatProfile 映射

### 2. 新增数据结构

| 结构体 | 用途 | 替代什么 |
|--------|------|----------|
| `DriveInputRampState` | 驾驶输入斜坡跨周期状态 | `ResolveDriveInput` 中的 static 局部变量 `keyboard_forward`/`keyboard_side` |
| `Dr16ModeResult` | DR16 拨杆解析结果 | `ResolveInputSemantics` 中 DR16 路径的局部变量 |
| `TcModeResult` | TC 键鼠解析结果 | `ResolveInputSemantics` 中 TC 路径的局部变量 |

### 3. 删除功能：手动倒地自启模式 (Recovery Manual Mode)

**GitHub 版本有**，本地版本**完全删除**：

- `TcSemanticState` 中删除：`recovery_manual_mode`、`z_recovery_armed`、`z_hold_ms`
- `ModeRequest` 中删除：`recovery_manual_mode`、`manual_left_leg_speed`、`manual_right_leg_speed`
- `ChassisFsmInput` 中删除：同上三个字段
- `Chassis::UpdateInput` 中删除：同上三个字段
- `chassis.cc` 中删除：手动模式 PID 控制逻辑（A/D/Ctrl+A/D 控制左右腿正反转）

Z 键长按 1s 切换倒地自启模式的功能也随之移除，Z 键现在只保留短按切换 AD 功能。

### 4. 删除功能：`dr16_parallel` 模式

`TcSemanticState.dr16_parallel` 字段被删除。该字段原来控制 DR16 键盘是否可以与 VT03 并行使用。

**影响**：`ResolveDriveInput` 的降级逻辑简化：
- GitHub：`(dr16_parallel && tc_remote.keyboard_value == 0) || tc_remote.tc_from_dr16`
- 本地：`tc_remote.tc_from_dr16`（仅在数据来自 DR16 回退时才降级到摇杆）

### 5. Static 变量 → 显式参数

| 位置 | GitHub (static) | 本地 (显式参数) |
|------|-----------------|-----------------|
| `ResolveDriveInput` | `static float keyboard_forward/side` | `DriveInputRampState &ramp` 参数 |
| `BuildChassisFsmInput` | `static uint32_t fall_start_ms` | 参数 `uint32_t& fall_start_ms` |
| `BuildChassisFsmInput` | `static bool was_posture_invalid` | 参数 `bool& was_posture_invalid` |
| 拨轮跳跃检测 | `static bool s_dial_jump_armed` | `TcSemanticState.dial_jump_armed` |
| 鼠标滚轮跳跃检测 | `static bool s_mouse_z_armed` | `TcSemanticState.mouse_z_jump_armed` |
| 使能边沿检测 | `static DomainRequest prev_domain` | `TcSemanticState.prev_domain` |

这消除了函数内 static 变量（对多实例/测试不友好），状态全部显式传递。

### 6. DR16 键盘位掩码：魔术数字 → 命名常量

- GitHub：`keys |= 0x0001; keys |= 0x0002; ...`
- 本地：`keys |= kRcKeyW; keys |= kRcKeyS; ...`

## 功能逻辑差异总结

| 行为 | GitHub | 本地 | 功能变化？ |
|------|--------|------|-----------|
| TC 键盘边沿检测 | 内联在 ResolveInputSemantics | 独立子函数 ResolveTcKeyboardEdges | **无** |
| DR16 拨杆映射 | 内联 | 独立子函数 ResolveDr16Mode | **无** |
| TC 键鼠映射 | 内联 | 独立子函数 ResolveTcMode | **无** |
| 云台目标积分 | 内联 | 独立子函数 IntegrateGimbalTarget | **无** |
| 自瞄模式判断 | 多处重复条件 | `IsAutoAimProfile()` 辅助函数 | **无** |
| Z 键长按切换手动恢复 | 有 | **删除** | **是** |
| A/D/Ctrl+A/D 手动腿速 | 有 | **删除** | **是** |
| dr16_parallel 降级逻辑 | `(parallel && kb==0) \|\| from_dr16` | `from_dr16` only | **是**（简化） |
| static 变量 | 6 处 static 局部变量 | 全部提取为参数/状态结构体 | **无**（行为相同） |
| DR16 键盘常量 | 魔术数字 | 命名常量 | **无** |

## 数据流路径（两种版本一致）

```
硬件 → SharedResources
  ├── dr16 (DR16遥控器)
  ├── gimbal_rx (云台CAN桥: VT03键鼠 + IMU)
  ├── aimbot (自瞄上位机)
  └── actuators (电机/IMU反馈)
        ↓
UpdateRawFeedbackAndInputSnapshot()
  ├── 读取 DR16 → Dr16RawInput
  ├── 读取 gimbal_rx → TcRemoteInput (VT03优先, 否则DR16回退)
  ├── 使能边沿检测 (disabled → enabled时重置状态)
  ├── ResolveInputSemantics() → ModeRequest
  │   ├── TC键盘边沿检测
  │   ├── DR16/TC模式解析
  │   ├── 自瞄覆盖
  │   ├── 组装ModeRequest字段
  │   └── 云台目标积分
  ├── 自瞄上位机目标更新
  └── 云台惯导数据
        ↓
BuildChassisFsmInput() → 底盘FSM输入
BuildGimbalFsmInput() → 云台FSM输入
        ↓
Chassis FSM / Gimbal FSM
```

## 结论

**本地版本是 GitHub 版本的重构 + 精简**：
1. 核心输入处理逻辑**功能等价**，只是拆分为更小的子函数
2. 删除了手动倒地自启模式（Z 长按 + A/D 操控腿速度）
3. 删除了 `dr16_parallel` 概念，简化了降级逻辑
4. 消除了所有 static 局部变量，改为显式状态传递
5. 代码风格改进（命名常量、辅助函数消除重复判断）
